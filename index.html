The Short Version

What is Claude?

Claude is an AI assistant from Anthropic. The chat is the part you have seen: ask a question, get a clear answer back. That is the floor, not the ceiling.

What sets it apart is everything around the chat. Claude can plug into your apps, take real actions, follow your house rules, and run jobs on a schedule. The rest of this deck is about turning it from a smart chatbot into something that does the work.

Getting Better Answers

Ask Like You Mean It

The quality of the answer tracks the quality of the ask. Give context, say what you are trying to do, and name the format you want back: a table, a draft email, five bullet points.

Treat it like a back-and-forth, not a search box. Push back, ask for a redo, paste in the real details. And start a new chat when you switch topics, so old context does not bleed into the next thing.

Claude Code · Staying in Control

You Decide How Much It Can Do

When Claude Code works on your computer, you are always the boss. There are just two controls to learn: the mode you put it in, and your answer each time it asks.

Control One

Pick a Mode

The mode sets how much Claude can do on its own. There are four modes you cycle through with Shift + Tab, plus Bypass permissions, a separate switch you turn on yourself. The tick shows which one is active.

Mode

Ask permissions1
Accept edits2
Plan mode3
Auto mode✓ 4
Bypass permissionsEnable

1 · Ask permissions  start here

Claude checks with you before it changes a file or runs anything. You approve each step. The safest.

2 · Accept edits

Claude edits files on its own, but still asks before bigger actions like running commands.

3 · Plan mode

Claude only looks and thinks. It writes out its plan and changes nothing until you approve it.

4 · Auto mode

Claude runs the whole task on its own, edits and commands, without stopping to ask. Keeps basic safety guardrails.

Bypass permissions

Skips every check, including the risky ones. Not in the normal cycle, you switch it on yourself. Leave it off until you truly know why you want it.

Coding · No Setup

Claude Code
in the Cloud

Claude Code also runs straight from your web browser or phone at claude.ai/code. Nothing to install. Claude does the work on its own computers in the cloud, then hands the result back to you.

The one thing it needs is a home for your code, and that home is GitHub. GitHub is the standard online locker where projects live. You connect your GitHub account once, and Claude works against the project you point it at.

Why GitHub?

Since your laptop is out of the picture, the code has to live somewhere Claude can reach. GitHub is that place, and it keeps a tidy history of every change so nothing is ever lost.

Claude Code · Web

How a cloud session works

1

Claude grabs a fresh copy of your project from GitHub.

2

It makes the changes you asked for on its own branch, a safe side-copy.

3

It pushes the work back to GitHub and can open a pull request for you to review.

4

You glance it over and approve. Your main project only changes when you say so.

Coding · Memory

CLAUDE.md:
A Note It Always Reads

A CLAUDE.md is a plain note that lives with your project. Claude Code reads it automatically at the start of every session, so it is the place to write down the things you would otherwise have to repeat each time.

House rules, how the project is laid out, the commands you use, your preferences in style and tone. Write it once, and Claude remembers it for good.

Why bother?

Less repeating yourself, and more consistent results. It is the single biggest thing you can do to make Claude Code feel like it already knows your project.

Two homes for your notes

Global · ~/.claude/CLAUDE.md

Rules for everything

Lives in your home folder and applies to every project. Best for personal preferences, like "always explain in plain English."

Project · ./CLAUDE.md

Rules for this project

Lives in the project folder and applies to just that one. Best for facts about this codebase. It can be shared with your team too.

Claude reads both, with the project note taking priority where they overlap.

Coding · One Command

/init: Let Claude
Write the First Draft

Staring at a blank CLAUDE.md is the hard part. You do not have to. Type /init in Claude Code and it will write the first draft for you.

Claude reads through your whole project, works out how it is built and which commands it uses, and saves a tidy CLAUDE.md in the folder. From there you just tweak it.

Run /init once on any project you work in often. Then add a line whenever you catch yourself explaining the same thing twice.

2

One thing to know about safety

Sensible habit: only connect apps you trust and actually need. You can switch any connector off at any time, or for one chat only.

Under the Hood

MCP:
The Common Language

Every connector you just saw speaks the same behind-the-scenes language, called MCP, short for Model Context Protocol. Think of it as a universal plug: instead of a different cable for every app, there is one shape that fits them all.

Because the plug is standard, anyone can build a new tool for Claude, and any app that follows the standard just works, without Claude needing custom code for each one.

1

You ask Claude to do something with one of your apps.

2

MCP carries the request to that app in a shape it understands.

3

The app answers, and Claude turns it back into plain language for you.

Why it matters to you

You never have to learn MCP. It just means the list of things Claude can plug into keeps growing, and they all work the same simple way.

The same safety rule still applies: a tool added through MCP only gets the access you approve, and you can switch it off whenever you like.
